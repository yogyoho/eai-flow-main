## 四、推荐策略：分层 Merge

鉴于 feature 分支与 2.0-rc 的冲突区域集中在 gateway 核心文件，建议采用**分层 merge** 策略，而非一次性 merge 或 rebase：

### 方案 A（推荐）：两步 Merge

```
Step 1: 先将 origin/main（= bytedance/main）合并进 feature/extensions-migration
  → 目标：feature 分支与上游 base 对齐（当前已处于此状态，无需操作）

Step 2: 从 bytedance/release/2.0-rc 逐模块合并
  → 拆分为 3 个子 merge，每个解决一个冲突域
```

#### Sub-step 2.1：合并认证与安全系统

```bash
# 创建临时分支
git checkout -b merge-2.0-rc-auth feature/extensions-migration

# 合并认证模块（新增文件为主，冲突最少）
git fetch bytedance
git merge bytedance/release/2.0-rc --no-commit \
  -- backend/app/gateway/auth/ \
     backend/app/gateway/auth_middleware.py \
     backend/app/gateway/csrf_middleware.py \
     backend/app/gateway/langgraph_auth.py \
     backend/app/gateway/internal_auth.py \
     backend/app/gateway/routers/auth.py \
     backend/app/gateway/routers/feedback.py
```

**预期冲突**：`backend/app/gateway/app.py`（注册新中间件）

**解决策略**：feature 分支的 `app.py` 中的扩展注册逻辑（如 `app.include_router(extensions_router)`）需要与 2.0-rc 的认证中间件共存。保留 feature 的扩展路由，追加 2.0-rc 的认证中间件链。

#### Sub-step 2.2：合并持久层

```bash
# 合并 persistence 模块
git merge bytedance/release/2.0-rc --no-commit \
  -- backend/packages/harness/deerflow/persistence/
```

**预期冲突**：无（纯新增目录）

#### Sub-step 2.3：合并网关路由和前端

```bash
# 合并核心 gateway 路由（高冲突区）
git merge bytedance/release/2.0-rc --no-commit \
  -- backend/app/gateway/routers/ \
     backend/app/gateway/services.py \
     frontend/src/app/workspace/ \
     frontend/src/app/\(auth\)/
```

**预期冲突**：
- `gateway/routers/threads.py` — feature 增加了用户隔离参数，2.0-rc 重写了路由签名
- `frontend/src/app/layout.tsx` — feature 增加了 AuthProvider，2.0-rc 增加了 (auth) 路由组
- `frontend/src/app/page.tsx` — feature 替换了 landing 组件

**解决策略**：
- `threads.py` — 以 feature 分支的用户隔离逻辑为基础，适配 2.0-rc 的新路由签名
- `layout.tsx` — 保留 feature 的 AuthProvider 和扩展，追加 2.0-rc 的 (auth) 路由组包裹
- `page.tsx` — feature 的 LandingNew 替换了老 landing，2.0-rc 也删除了老组件 → **无冲突，保留 feature 版本**

### 方案 B：Rebase + 手动 Patch（更精细控制）

```bash
# 1. 在 feature/extensions-migration 上创建补丁
git format-patch origin/main --stdout > feature_extensions.patch

# 2. 切到 bytedance/release/2.0-rc
git checkout bytedance/release/2.0-rc
git checkout -b feature-onto-2.0-rc

# 3. 尝试应用 feature 分支的补丁
git am --reject < feature_extensions.patch
# --reject 会生成 .rej 文件标记未成功合并的 hunk

# 4. 手动解决每个 .rej 文件
# 5. 完成后验证测试通过
```

## 六、执行脚本模板

### 自动生成 patch 文件

```bash
#!/bin/bash
# generate_patches.sh - 生成各模块的 patch 文件

FEATURE_BRANCH="feature/extensions-migration"
UPSTREAM_2_0_RC="bytedance/release/2.0-rc"
PATCH_DIR="./manual_patches"

mkdir -p "$PATCH_DIR"

# 1. 生成 feature 分支相比 origin/main 的补丁
git format-patch origin/main --stdout > "$PATCH_DIR/feature_extensions.patch"

# 2. 生成 2.0-rc 相比 bytedance/main 的补丁
git format-patch bytedance/main --stdout \
  --爵 bytedance/release/2.0-rc > "$PATCH_DIR/upstream_2.0-rc.patch"

# 3. 生成各模块的差异文件
echo "=== Auth module ===" > "$PATCH_DIR/module_diffs.txt"
git diff origin/main.."$UPSTREAM_2_0_RC" -- \
  backend/app/gateway/auth/ \
  backend/app/gateway/*middleware.py \
  backend/app/gateway/routers/auth.py \
  >> "$PATCH_DIR/module_diffs.txt"

echo "=== Persistence module ===" >> "$PATCH_DIR/module_diffs.txt"
git diff origin/main.."$UPSTREAM_2_0_RC" -- \
  backend/packages/harness/deerflow/persistence/ \
  >> "$PATCH_DIR/module_diffs.txt"

echo "=== Gateway routers ===" >> "$PATCH_DIR/module_diffs.txt"
git diff origin/main.."$UPSTREAM_2_0_RC" -- \
  backend/app/gateway/routers/ \
  >> "$PATCH_DIR/module_diffs.txt"

echo "Patches generated in $PATCH_DIR/"
```

### 分步 Merge 执行脚本

```bash
#!/bin/bash
# merge_2.0-rc.sh - 分步将 2.0-rc 合并到 feature 分支

set -e
FEATURE_BRANCH="feature/extensions-migration"
UPSTREAM="bytedance/release/2.0-rc"

echo "[1/4] Checking out feature branch..."
git checkout "$FEATURE_BRANCH"
git pull origin "$FEATURE_BRANCH"

echo "[2/4] Merging auth module..."
git merge --no-commit --no-ff "$UPSTREAM" \
  -- backend/app/gateway/auth/ \
     backend/app/gateway/*middleware.py \
     backend/app/gateway/routers/auth.py \
     backend/app/gateway/routers/feedback.py
# 解决 app.py 冲突后：
git add backend/app/gateway/app.py
git commit -m "Merge auth module from $UPSTREAM"

echo "[3/4] Merging persistence module..."
git merge --no-commit --no-ff "$UPSTREAM" \
  -- backend/packages/harness/deerflow/persistence/
git commit -m "Merge persistence module from $UPSTREAM"

echo "[4/4] Merging gateway routers..."
git merge --no-commit --no-ff "$UPSTREAM" \
  -- backend/app/gateway/routers/ \
     backend/app/gateway/services.py
# 解决 threads.py, memory.py, services.py 冲突后：
git add -A
git commit -m "Merge gateway routers from $UPSTREAM"

echo "Merge complete! Run tests:"
echo "  cd backend && python -m pytest tests/ -v"
echo "  cd frontend && pnpm test"
```
