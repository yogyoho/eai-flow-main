## 三、发现的问题及修复状态

### BUG-1: 自定义角色缺少 `system:access` ✅ 已修复

- **严重性**: 高
- **根因**: 角色创建时未自动包含 `system:access` 权限
- **修复**: `RoleService.create_role()` 现在自动合并 `BASE_PERMISSIONS = {system:access, kb:read, doc:read}`
- **状态**: ✅ 代码层面已修复

### BUG-2: 审批提交 500 错误 ✅ 已修复

- **严重性**: 高
- **根因**: `approval_workflows` 表缺少 `reviewer_id` 列 — SQLAlchemy 模型定义了但 CREATE TABLE 迁移未包含
- **修复**: 1) database.py CREATE TABLE 添加 reviewer_id 列; 2) ALTER TABLE IF NOT EXISTS 兼容已有数据库
- **验证**: ✅ 提交审批返回 200, step_count=2

### BUG-3: 用户列表显示旧部门名 ⚠️ 低优先级

- **严重性**: 低
- **根因**: 代码逻辑正确(update_user 已有 db.refresh)，可能是 SQLAlchemy 会话缓存或前端刷新问题
- **状态**: 暂不处理，不影响核心功能

### GAP-1: Markdown 直接生成 Word 不支持 ✅ 已修复

- **根因**: 输出端点只支持文件上传，不支持直接传入 content 字符串
- **修复**: 添加 `content` Form 字段，与 `file` 二选一
- **验证**: ✅ POST /generate source=markdown&content=... 返回 200 + download_url

### GAP-2: Dashboard 不显示未"进入"项目的成员任务 ⚠️ 待处理

- **现象**: 王二被添加到项目但 Dashboard 显示"暂无待办任务"
- **根因**: my-tasks API 可能依赖 thread 绑定
- **建议**: 项目成员应自动显示待办任务
