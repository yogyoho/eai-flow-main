# 固体矿产地质勘查报告制作技能设计

**日期**: 2026-05-18
**状态**: 已批准
**形态**: SKILL.md 智能体技能（单一全面方案）

## 1. 概述

### 1.1 目标

为 DeerFlow 智能体系统创建一个地质勘查报告制作技能，支持生成符合中国国家标准的固体矿产地质勘查报告。技能以 SKILL.md 形式存在于 `skills/custom/geological-report/`，通过系统提示词注入引导智能体完成报告编写全流程。放置在 `custom/` 目录下，便于用户编辑和迭代，部署时需注意 custom 目录的持久化（该目录默认被 gitignore）。

### 1.2 适用标准

| 标准 | 名称 | 用途 |
|------|------|------|
| DZ/T 0033-2020 | 固体矿产地质勘查报告编写规范 | 报告结构模板、章节要求 |
| GB/T 13908-2020 | 固体矿产地质勘查规范 总则 | 勘查程度、工程间距、资源量分类 |
| GB/T 33444-2016 | 固体矿产勘查工作规范 | 勘查工作程度、质量要求 |

### 1.3 矿种覆盖

- 主覆盖：通用固体矿产（三标准通用条款）
- 矿种适配提示：煤（DZ/T 0215-2020）、铜铅锌银镍钼（DZ/T 0214-2020）
- 其他矿种：引导用户提供矿种专用工业指标，使用通用框架

### 1.4 设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 技能形态 | SKILL.md | 用户选择，纯提示词驱动 |
| 架构方案 | 单一全面 SKILL | 三核心标准方案知识量可控 |
| 勘查阶段 | 普查+详查+勘探 | 完整覆盖三阶段 |
| 矿种支持 | 多矿种定制 | 按矿种调整工业指标和特殊章节 |
| 输入方式 | 双模式（数据文件解析+对话引导） | 兼顾有数据和无数据场景 |
| 输出格式 | Markdown + 可迭代修改 | 灵活输出，支持后续转换 |

## 2. 技能结构

### 2.1 文件位置

```
skills/custom/geological-report/
└── SKILL.md
```

> **注意**: `skills/custom/` 目录默认被 `.gitignore` 排除。技能放置在此目录后会被系统自动发现（`SkillCategory.CUSTOM`），但需要在 `extensions_config.json` 中手动启用：
> ```json
> { "skills": { "geological-report": { "enabled": true } } }
> ```
> 部署时需确保 custom 目录的持久化存储。

### 2.2 SKILL.md Frontmatter

```yaml
---
name: geological-report
description: 固体矿产地质勘查报告制作技能 — 基于 DZ/T 0033-2020、GB/T 13908-2020、GB/T 33444-2016 标准，支持普查/详查/勘探三阶段，多矿种定制，双模式输入（数据文件解析 + 对话引导），生成符合国家标准的 Markdown 格式报告
license: MIT
allowed-tools: ["bash", "read_file", "write_file", "str_replace", "ls", "ask_clarification"]
---
```

### 2.3 SKILL.md 章节组织

1. **概述与角色** — 技能身份、适用标准范围、矿种范围
2. **工作流协议** — 双模式入口、阶段判定、报告生成、迭代修改
3. **阶段报告模板** — 普查(附录A)、详查(附录B)、勘探(附录C) 三套完整目录和各章节写作指导
4. **技术要求精要** — GB/T 13908-2020 各阶段勘查程度、勘查类型划分、工程间距、采样要求
5. **资源量分类体系** — TD/KZ/TM/KX/ZS 分类定义和使用规则
6. **质量校验清单** — 生成后自检：章节完整性、数据一致性、术语规范性
7. **矿种适配引导** — 煤、铜铅锌银镍钼的关键差异点和工业指标
8. **输出规范** — Markdown 格式要求、文件命名、章节编号规则

## 3. 工作流协议

### 3.1 入口判定

```
技能激活时:
  IF 用户上传了数据文件（Excel/CSV/PDF/Word）:
      → 自动解析模式
  ELSE:
      → 对话引导模式
```

### 3.2 自动解析模式

1. 使用 `read_file` 和 `bash`（Python 脚本）解析上传文件
2. 提取关键信息：勘查区位置、矿种、勘查阶段、矿体参数、样品分析数据、工程布置
3. 将提取结果结构化呈现给用户确认
4. 确认后进入报告生成阶段

### 3.3 对话引导模式

按优先级依次收集信息（用户已有信息可跳过）：

| 序号 | 信息类别 | 关键字段 |
|------|----------|----------|
| 1 | 矿种类型 | 煤/铜/铅/锌/银/镍/钼/其他 |
| 2 | 勘查阶段 | 普查/详查/勘探 |
| 3 | 项目基本信息 | 项目名称、地理位置、勘查面积、勘查单位、工作期限 |
| 4 | 区域地质概况 | 地层、构造、岩浆岩、变质岩 |
| 5 | 矿区地质 | 地层、构造、岩浆岩、蚀变带、地球化学异常 |
| 6 | 矿体描述 | 矿体编号、形态、产状、规模（走向/倾向/延深）、厚度、品位 |
| 7 | 勘查工作 | 钻探/槽探/坑探工程量、工程质量评述 |
| 8 | 样品与化验 | 采样方法、分析项目、内检外检结果 |
| 9 | 开采技术条件 | 水文地质、工程地质、环境地质（详查/勘探阶段） |
| 10 | 资源量估算 | 工业指标、估算方法、块段划分、估算结果 |

### 3.4 报告生成

1. 根据勘查阶段选择对应模板（附录A/B/C）
2. 逐章生成 Markdown 内容
3. 使用 `write_file` 写入 `/mnt/user-data/outputs/{project-name}-report.md`
4. 每完成一个主要章节向用户汇报进度

### 3.5 质量自检

报告完成后，智能体执行以下检查：

**完整性检查**：
- 当前阶段模板的所有必填章节是否都有内容
- 每个章节内的必填子项是否覆盖

**数据一致性检查**：
- 矿体参数（走向、倾向、倾角、厚度、品位）在各章节引用时一致
- 工程量统计数字与正文描述吻合
- 资源量估算表与正文数据对应
- 术语使用统一

**标准符合性检查**：
- 资源量分类使用正确术语（TD/KZ/TM/KX/ZS）
- 工程间距符合勘查类型对应要求
- 采样质量评述包含必要质量指标

### 3.6 迭代修改

用户指出需要修改的部分，智能体：
1. 定位需要修改的章节
2. 修改内容
3. 检查修改是否影响其他章节的一致性
4. 更新文件

## 5. 技术要求精要

### 5.1 勘查类型划分（GB/T 13908-2020）

| 类型 | 代号 | 特征 |
|------|------|------|
| 简单 | Ⅰ | 矿体规模大、形态简单、品位分布均匀 |
| 中等 | Ⅱ | 矿体规模中等、形态较简单、品位较均匀 |
| 复杂 | Ⅲ | 矿体规模小、形态复杂、品位变化大 |
| 过渡类型 | Ⅰ-Ⅱ / Ⅱ-Ⅲ | 兼具相邻类型特征 |

### 5.2 工程间距参考

按勘查类型和阶段给出推荐间距范围，具体数值由矿种规范和实际情况确定。SKILL.md 内嵌通用参考表，并引导用户确认。

### 5.3 资源量分类体系

| 分类 | 代号 | 全称 | 含义 |
|------|------|------|------|
| 探明资源量 | TM | 探明的 | 工程控制程度最高，矿体形态、产状、品位已详细查明 |
| 控制资源量 | KZ | 控制的 | 工程控制程度较高，矿体主要特征已基本查明 |
| 推断资源量 | TD | 推定的 | 工程控制程度较低，矿体总体特征初步查明 |
| 可信储量 | KX | 可信的 | 经过预可行性或可行性研究的控制资源量/探明资源量 |
| 证实储量 | ZS | 证实的 | 经过可行性研究的探明资源量 |

### 5.4 各阶段勘查程度要求

**普查阶段**：
- 地质填图 1:25000～1:5000
- 初步查明矿体规模、形态、产状
- 推断资源量为主
- 矿种工业指标参考一般工业指标

**详查阶段**：
- 地质填图 1:10000～1:2000
- 基本查明矿体特征、矿石质量
- 控制资源量+推断资源量
- 矿石加工选冶技术性能初步评价

**勘探阶段**：
- 地质填图 1:5000～1:500
- 详细查明矿体形态、产状、品位变化
- 探明资源量+控制资源量+推断资源量
- 可行性评价

## 6. 矿种适配引导

### 6.1 煤矿（参照 DZ/T 0215-2020）

**关键差异**：
- 工业指标：最低可采厚度、最高灰分、最低发热量、最高硫分
- 特殊章节：煤层对比与可靠性分析、煤质评价（工业分析、元素分析、工艺性能）
- 资源量分类使用煤炭专用标准
- 2020版已移除泥炭相关内容

### 6.2 铜、铅、锌、银、镍、钼（参照 DZ/T 0214-2020）

**关键差异**：
- 工业指标：边界品位、最低工业品位、最小可采厚度、夹石剔除厚度
- 基本分析项目按矿种列出（如 Cu: Cu、Ag、Au、S、As；Zn: Zn、Pb、Ag、S、Cd）
- 伴生元素综合评价要求
- 绿色勘查要求（2020版新增）

### 6.3 其他矿种

- 引导用户提供该矿种的具体工业指标
- 使用 GB/T 13908 通用框架生成报告
- 提示用户参照对应矿种专用规范

## 7. 输出规范

### 7.1 文件格式

- 格式：Markdown（.md）
- 编码：UTF-8
- 命名：`{项目名称}-{阶段}-地质勘查报告.md`（如 `张家沟-勘探-地质勘查报告.md`）
- 存储路径：`/mnt/user-data/outputs/`

### 7.2 章节编号

- 一级章节：`第一章`、`第二章`...
- 二级章节：`1.1`、`1.2`...
- 三级章节：`1.1.1`、`1.1.2`...

### 7.3 表格格式

Markdown 表格，关键数据表（如资源量估算结果表）使用以下格式：

```markdown
| 块段编号 | 矿体编号 | 面积(m²) | 平均厚度(m) | 体积(m³) | 矿石量(t) | 平均品位 | 金属量(t) | 资源量分类 |
|----------|----------|----------|-------------|----------|-----------|----------|-----------|-----------|
```

### 7.4 图表描述

对于无法直接生成的图表（勘探线剖面图、矿体投影图等），生成文字描述块：

```markdown
> **[图表: {图表名称}]**
> - 类型: {剖面图/投影图/等值线图等}
> - 比例尺: {1:XXXX}
> - 内容描述: {详细描述应展示的内容}
> - 数据来源: {引用数据出处}
```

## 8. 实现范围

### 8.1 包含

- 完整的 SKILL.md 文件，包含所有标准知识、模板和工作流
- 三阶段报告模板（附录A/B/C 结构）
- GB/T 13908-2020 技术要求精要表
- 资源量分类体系
- 质量校验清单
- 煤矿和铜铅锌银镍钼矿种适配引导

### 8.2 不包含

- 图形生成能力（剖面图、平面图等需专业软件）
- 数据库或 GIS 数据接口
- 后端 API 或前端页面（纯 SKILL.md）
- 矿种专用规范的完整内容（仅嵌入关键差异点）
- 资源量估算数值计算（智能体辅助但不替代专业软件）
