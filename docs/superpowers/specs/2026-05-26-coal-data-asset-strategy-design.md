# 煤炭行业数据资产建设与变现策略设计

> **日期**: 2026-05-26
> **状态**: 待实施
> **范围**: 煤炭招投标/供应链平台数据资产化与变现

---

## 1. 背景与目标

煤炭领域招投标和供应链互联网平台积累了海量数据，包括供应商、招标、采购、财务、供应链等数据。本设计定义如何将这些数据构建为高质量数据集，并通过 MCP 服务订阅、数据集售卖、AI 训练平台三种模式实现数据资产变现。

### 现状

- **数据存储**: 混合存储（结构化数据库 + 非结构化文档）
- **数据量级**: 100万-1000万条记录 + 大量 PDF/Word 文档
- **变现方向**: AI 数据服务（MCP/API）为核心，数据集和训练平台为辅
- **策略**: 渐进式——先嵌入 EAI Flow 平台验证，后独立扩展

## 2. 数据集建设架构

### 2.1 数据分层架构

```
原始数据层 → 清洗标注层 → 合规脱敏层 → 结构化数据层 → 服务发布层
    ↓             ↓             ↓              ↓              ↓
  数据采集      质量控制       安全合规       数据资产目录     MCP/API/数据集
                                                        ↕
                                                    实时数据流
```

### 2.3 清洗标注层

1. **文档智能抽取**: AI 从 PDF/Word 提取结构化字段（项目名称、金额、技术参数、资质要求、煤质指标等）
2. **实体标准化**: 统一供应商名称（去除别名/曾用名）、煤种分类（国标编码）、地区编码、度量单位
3. **关联图谱构建**: 供应商-项目-中标-交付-付款 完整关系链，支持图谱查询
4. **质量评分**: 每条数据计算完整性、准确性、时效性三维评分，标注数据可信度
5. **实时流处理**: 对接实时数据源（价格行情、物流轨迹、库存变动），通过流式管道实时入库

### 2.4 合规脱敏层

1. **敏感数据识别**: 自动识别商业秘密（报价、利润率）、个人隐私（联系人信息）、价格敏感信息
2. **分级脱敏**: 根据订阅等级决定是否展示原始价格、供应商联系方式等敏感字段
3. **合规审计日志**: 记录每次数据访问的调用方、查询内容、返回数据范围
4. **数据水印**: 返回数据中嵌入调用方标识水印，防止数据转售和泄露追溯
5. **行业合规**: 满足煤炭行业监管要求（数据出境限制、国企数据管理规范等）

### 2.5 结构化数据层（数据资产目录）

| 数据集名称 | 核心字段 | 更新频率 | 数据量级 |
|-----------|---------|---------|---------|
| 供应商画像 | 资质、业绩、信用、履约、地域 | 周更 | 50,000+ |
| 招投标分析 | 项目趋势、中标价、竞争分布 | 日更 | 200,000+ |
| 价格指数 | 各煤种价格、区域价差、季节波动 | 实时 | 100,000+ |
| 供应链风险 | 集中度、替代方案、瓶颈、政策 | 周更 | 持续累积 |
| 行业知识 | 煤炭分类、检测参数、法规政策 | 月更 | 100,000+ 节点 |
| 质量检测 | 热值、灰分、硫分、挥发分 | 日更 | 500,000+ |
| 产能库存 | 矿区/港口产能、库存、物流状态 | 实时 | 持续累积 |
| 实时行情 | 现货/期货价格、运费、供需指数 | 实时 | 持续累积 |

## 3. 变现模式 A：MCP 数据服务

### 3.1 MCP Tool 设计

| Tool | 输入参数 | 返回内容 | 定价等级 |
|------|---------|---------|---------|
| `supplier_search` | 关键词/地区/资质/评级 | 供应商画像列表 | 基础 |
| `supplier_risk_assessment` | 供应商ID | 风险评分+详情 | 高级 |
| `bidding_trend_analysis` | 时间范围/地区/煤种 | 中标趋势、价格走势 | 基础 |
| `bidding_competitor_analysis` | 项目关键词 | 竞争格局、中标概率预测 | 高级 |
| `price_index_query` | 煤种/地区/时间范围 | 价格指数+历史曲线 | 基础 |
| `price_forecast` | 煤种/预测周期 | 价格预测+置信区间 | 专业 |
| `supply_chain_risk` | 供应链节点 | 断供风险、替代方案 | 高级 |
| `coal_quality_search` | 矿区/煤种/参数范围 | 煤质检测数据 | 基础 |
| `capacity_inventory` | 地区/时间范围 | 产能+库存数据 | 专业 |
| `market_report_generate` | 主题/时间范围 | 自动化市场分析报告 | 专业 |
| `realtime_price_stream` | 煤种/订阅条件 | 实时价格推送(SSE) | 专业 |

### 3.2 订阅等级

| 等级 | 月费 | 可用工具 | 日调用上限 | 特性 |
|------|------|---------|-----------|------|
| 基础版 | ¥2,000 | supplier_search, bidding_trend, price_index, coal_quality | 1,000 | 标准脱敏 |
| 高级版 | ¥8,000 | 基础版 + risk_assessment, competitor, supply_chain_risk | 5,000 | 细粒度脱敏 |
| 专业版 | ¥25,000 | 全部工具 + 预测 + 实时推送 + 报告生成 | 无限 | 完整数据 |
| 企业版 | 定制 | 全部 + 私有部署 + 专属数据源 | 自定义 | SLA 保障 |

### 3.3 技术实现（嵌入 EAI Flow）

```
backend/app/extensions/coal_data/
├── pipelines/                    # 数据抽取管道
│   ├── document_extractor.py     # PDF/Word 文档智能抽取
│   ├── entity_normalizer.py      # 实体标准化
│   ├── quality_scorer.py         # 数据质量评分
│   └── realtime_pipeline.py      # 实时数据流处理
├── datasets/                     # 数据集管理
│   ├── supplier.py               # 供应商画像
│   ├── bidding.py                # 招投标分析
│   ├── price_index.py            # 价格指数
│   ├── quality_test.py           # 煤质检测
│   ├── supply_chain.py           # 供应链数据
│   └── knowledge.py              # 行业知识
├── compliance/                   # 合规脱敏
│   ├── desensitizer.py           # 脱敏处理
│   ├── watermark.py              # 数据水印
│   └── audit_logger.py           # 审计日志
├── mcp_server/
│   └── coal_data_server.py       # 煤炭数据 MCP Server
├── billing/                      # 计费系统
│   ├── subscription.py           # 订阅管理
│   ├── quota.py                  # 调用配额
│   └── metering.py               # 用量计量
├── routers.py                    # API 路由
├── service.py                    # 业务逻辑
└── schemas.py                    # 数据模型
```

MCP Server 注册到 `extensions_config.json`，AI Agent 自动获得数据查询能力。外部应用通过标准 MCP 协议接入。

## 8. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 数据合规风险 | 数据泄露、违规罚款 | 合规脱敏层 + 法律审查 + 数据水印 |

| 客户教育成本高 | 转化率低 | 免费数据集引流 + MCP 体验券 + 行业案例 |
