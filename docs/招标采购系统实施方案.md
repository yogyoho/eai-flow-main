### 2.2 创建报告类型模板（Report Type Templates）

通过 Knowledge Factory Pipeline 上传历史招标文件生成模板：

```bash
# 步骤1：上传源报告（历史招标文件 -> KnowledgeBase documents）
POST /api/extensions/knowledge-bases/{kb_id}/documents  # 上传到 KB-2

# 步骤2：创建抽取任务
POST /api/extensions/kf/tasks
{
  "source_report_ids": ["doc_id_1", "doc_id_2", "doc_id_3"],
  "domain": "bidding_procurement",
  "name": "工程类招标文件模板",
  "config": {
    "llm_model": "qwen-max",
    "chunk_strategy": "semantic",
    "merge_threshold": 0.85
  }
}

# Pipeline 自动执行 5 个步骤：
#   Step1: 文档解析（RAGFlow API）
#   Step2: LLM 推断章节结构（生成标准招标文件章节树）
#   Step3: LLM 逐节抽取内容契约（每章 key_elements, structure_type, forbidden_phrases）
#   Step4: 多报告去重合并
#   Step5: 合规规则引擎校验

# 重复上述步骤，为每种项目类型生成模板：
#   - 工程类招标文件模板
#   - 货物类招标文件模板
#   - 服务类招标文件模板
#   - 评标报告模板
#   - 合同模板
```

## 七、11 个场景逐项实施步骤

### 场景 4: 招标方案审核

```
触发方式：用户上传招标方案 → Agent 调用 bidding-doc-reviewer skill

数据流：
  1. 解析上传的招标方案文档 → Knowledge Factory Step1 (RAGFlow解析)
  2. 获取数据结构：Lead Agent 分析文档
  3. 加载合规规则：
     match_rules.py --industry bidding_procurement --report-type bidding_plan
  4. 执行多维度检测：
     ├── 完整性检查 (StandardCheckValidator)
     ├── 资质条件合理性 (ComplianceCheckValidator)
     ├── 排斥限制条款检测 (自定义 forbidden_patterns)
     ├── 流程合规性 (ComplianceCheckValidator)
     └── 错敏词扫描 (forbidden_phrases 匹配)
  5. 生成审查报告：含问题分级 + 具体位置 + 法规依据 + 修改建议
```

### 场景 6: 招标文件检测

```
触发方式：用户上传招标文件 → Agent 调用 bidding-doc-reviewer skill

数据流：
  1. RAGFlow 解析招标文件 → 结构化数据
  2. 加载全部招标文件合规规则（~30条）
  3. 执行多维度检测：
     ├── 合规性检测（法规引用、强制招标、排斥限制）
     ├── 合理性检测（资质/业绩/财务要求与项目规模匹配）
     ├── 错敏词检测（200+ 禁用词库）
     └── 结构完整性检测（必含章节）
  4. 输出"体检报告"：
     - 问题总数、按严重级别分类
     - 每个问题：位置定位、违规依据、修改建议
     - 合规率百分比
```

### 场景 8: 投标合规自查

```
触发方式：用户上传招标文件 + 投标文件 → Agent 调用 bid-response-checker skill

数据流：
  1. 并行解析两份文档
  2. 从招标文件中提取全部要求条款（技术+商务+资格）
  3. 逐条比对投标文件的响应情况
  4. 专项检查：
     ├── 资质证书有效期
     ├── 业绩证明完整性
     ├── 签字盖章规范性
     ├── 报价完整性 + 低于成本价预警（<控制价80%）
     └── 分项报价与总价一致性
  5. 输出响应性对照表（响应/部分响应/未响应）+ 修改建议
```

### 场景 11: 智能辅助评标

```
触发方式：用户上传招标文件 + 多份投标文件 → evaluation-assistant skill

数据流：
  1. 多模态提取各投标文件关键要素（Knowledge Factory Pipeline）
  2. 结构化对照：
     ├── 技术参数横向对比表
     ├── 商务条款响应对比
     └── 报价清单对比
  3. 基于评分标准计算建议评分
  4. 异常检测：
     ├── 投标文件雷同度分析
     └── 报价异常偏离检测
  5. 输出辅助评审结果：对比表 + 评分建议 + 异常提示
```

### 场景 12: 评标报告核验

```
触发方式：用户上传评标报告 → Compliance Engine

数据流：
  1. 解析评标报告 → 提取评分数据、评审结论、候选推荐
  2. 数据一致性检查：
     ├── 总分 vs 分项之和 (DataConsistencyValidator, threshold=0.01)
     ├── 各专家评分 vs 最终得分
     └── 计算过程逐项复核
  3. 程序合规检查：
     ├── 候选人数量合规（1-3名）
     ├── 废标理由合理性
     └── 签章完整性
  4. 专家打分偏离度检查：
     ├── 各专家打分 vs 均值偏离 > 20% → 预警
     └── 特定投标人打分异常 → 预警
  5. 输出：问题清单 + 位置定位 + 修正建议
```
