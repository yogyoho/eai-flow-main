# EAI-Flow (DeerFlow 2.0) 系统模块功能分析

> 本文档基于 D:\eai\eai-flow-main 代码库分析，覆盖 EAI-Flow 系统（基于 ByteDance DeerFlow 2.0 的增强版）

---

## 一、系统架构总览

### 1.1 运行时拓扑

```
浏览器 → Nginx (统一入口 4026)
  ├── /api/langgraph/* → Gateway 嵌入式 LangGraph 运行时 (4001)，URL 重写为 /api/*
  ├── /api/*           → Gateway REST API (4001)
  └── /*               → Next.js 前端 (4000)
```

### 1.2 服务端口（本地开发，非 Docker）

| 服务 | 端口 | 技术栈 |
|------|------|--------|
| nginx（统一入口） | 4026 | nginx |
| Gateway API（FastAPI） | 4001 | Python 3.12+, FastAPI, LangGraph |
| LangGraph 运行时 | 4024 | LangGraph (嵌入式，非独立服务) |
| 前端（Next.js） | 4000 | Next.js 16, React 19, TypeScript |

### 1.3 代码分层

```
eai-flow-main/
├── backend/                        # Python 后端
│   ├── packages/harness/deerflow/  # Harness 框架层 (deerflow.* 可发布包)
│   └── app/                        # App 应用层 (app.*)
│       ├── gateway/                # FastAPI Gateway API
│       ├── channels/               # IM 渠道集成
│       └── extensions/             # 企业扩展模块 (EAI-Flow 独有)
├── frontend/                       # Next.js 前端
│   └── src/
│       ├── app/                    # 页面路由
│       ├── components/             # React 组件
│       ├── core/                   # 核心业务逻辑
│       └── extensions/             # 前端扩展模块
├── skills/                         # Agent 技能包
│   ├── public/                     # 内置技能
│   └── custom/                     # 自定义技能
├── docker/                         # Docker 编排
├── scripts/                        # 运维脚本
└── docs/                           # 设计文档
```

## 二、后端 Harness 框架层 (`backend/packages/harness/deerflow/`)

### 2.1 Agent 系统 (`agents/`)

| 组件 | 文件 | 功能描述 | 技术参数 |
|------|------|----------|----------|
| **Lead Agent 工厂** | `agents/lead_agent/agent.py` | LangGraph 图入口 `make_lead_agent()`；动态模型选型、工具加载、系统提示词组装 | 注册于 `langgraph.json`，接受 `RunnableConfig` |
| **ThreadState** | `agents/thread_state.py` | 扩展 `AgentState` 的运行状态，包含 sandbox、thread_data、title、artifacts、todos、uploaded_files、viewed_images | 使用自定义 reducer: `merge_artifacts`, `merge_viewed_images` |
| **运行时配置** | `config.configurable` | `thinking_enabled`, `model_name`, `is_plan_mode`, `subagent_enabled` | 通过 `RunnableConfig` 传递 |

#### 中间件链（18 个，严格顺序）

| # | 中间件 | 功能 | 可选/必选 | 配置项 |
|---|--------|------|-----------|--------|
| 1 | **ThreadDataMiddleware** | 创建每线程隔离目录 `users/{user_id}/threads/{thread_id}/user-data/{workspace,uploads,outputs}` | 必选 | — |
| 2 | **UploadsMiddleware** | 追踪并注入新上传文件到对话 | 必选 | — |
| 3 | **SandboxMiddleware** | 获取 sandbox 环境，存储 `sandbox_id` | 必选 | — |
| 4 | **DanglingToolCallMiddleware** | 为被中断的工具调用注入占位 ToolMessage | 必选 | — |
| 5 | **LLMErrorHandlingMiddleware** | Provider 调用异常归一化 | 必选 | — |
| 6 | **GuardrailMiddleware** | 工具调用预授权检查 | 可选 | `guardrails.enabled: true` |
| 7 | **SandboxAuditMiddleware** | Sandbox 操作安全审计日志 | 可选 | — |
| 8 | **ToolErrorHandlingMiddleware** | 工具异常转换为 ToolMessage | 必选 | — |
| 9 | **SummarizationMiddleware** | 接近 token 限制时压缩上下文 | 可选 | `summarization.enabled` |
| 10 | **TodoListMiddleware** | 计划模式任务跟踪 (`write_todos` tool) | 可选 | `is_plan_mode: true` |
| 11 | **TokenUsageMiddleware** | 记录 token 使用量 | 可选 | `token_usage.enabled` |
| 12 | **TitleMiddleware** | 首轮对话后自动生成标题 | 可选 | `title.enabled` |
| 13 | **MemoryMiddleware** | 异步记忆提取 (过滤 user + final AI) | 可选 | `memory.enabled` |
| 14 | **ViewImageMiddleware** | 注入 base64 图片数据 | 条件 | 模型支持 vision |
| 15 | **DeferredToolFilterMiddleware** | 隐藏延迟工具 schema | 可选 | — |
| 16 | **SubagentLimitMiddleware** | 截断多余 `task` 工具调用 | 可选 | `subagent_enabled` |
| 17 | **LoopDetectionMiddleware** | 检测重复工具调用循环 | 必选 | — |
| 18 | **ClarificationMiddleware** | 拦截 `ask_clarification` → `Command(goto=END)` | 必选 | 必须最后 |

### 2.2 记忆系统 (`agents/memory/`)

| 组件 | 文件 | 功能 | 技术参数 |
|------|------|------|----------|
| **Updater** | `updater.py` | LLM 驱动的记忆提取、事实去重 | 去重方式：trim whitespace 后比较 |
| **Queue** | `queue.py` | 防抖更新队列 | 默认 30s 等待，per-thread 去重 |
| **Storage** | `storage.py` | 文件存储，per-user 隔离 | 路径: `{base_dir}/users/{user_id}/memory.json` |
| **Prompt** | `prompt.py` | 记忆提取提示词模板 | — |

**数据结构**：
- User Context: `workContext`, `personalContext`, `topOfMind` (1-3句摘要)
- History: `recentMonths`, `earlierContext`, `longTermBackground`
- Facts: `{id, content, category, confidence(0-1), createdAt, source}`
- 配置: `max_facts=100`, `fact_confidence_threshold=0.7`, `max_injection_tokens=2000`

### 2.3 Sandbox 系统 (`sandbox/`)

| 组件 | 功能 | 技术参数 |
|------|------|----------|
| **Sandbox (抽象接口)** | `execute_command`, `read_file`, `write_file`, `list_dir` | 线程隔离 |
| **LocalSandboxProvider** | 本地文件系统执行 | bash 默认禁用；per-thread LRU 缓存 (256)；路径映射 `/mnt/user-data/*` → host 目录 |
| **AioSandboxProvider** | Docker 容器隔离 | 位于 `community/`；容器内 `/mnt/user-data/*` volume-mount |
| **Sandbox Tools** | bash, ls, read_file, write_file, str_replace | str_replace 按 `(sandbox.id, path)` 序列化 |

**虚拟路径映射**：
| 虚拟路径 | 用途 | 物理路径映射 |
|----------|------|-------------|
| `/mnt/user-data/workspace/` | Agent 工作目录 | `backend/.deer-flow/users/{uid}/threads/{tid}/user-data/workspace/` |
| `/mnt/user-data/uploads/` | 上传文件 | 同上 uploads/ |
| `/mnt/user-data/outputs/` | 输出文件 | 同上 outputs/ |
| `/mnt/skills/` | 技能包 | `deer-flow/skills/` |
| `/mnt/acp-workspace/` | ACP agent 工作区 | `{base}/users/{uid}/threads/{tid}/acp-workspace/` (read-only) |

### 2.4 子 Agent 系统 (`subagents/`)

| 组件 | 功能 | 技术参数 |
|------|------|----------|
| **内置 Agent** | `general-purpose` (全工具集), `bash` (命令专家) | — |
| **执行引擎** | `SubagentExecutor` 后台线程执行 | 调度池 3 workers + 执行池 3 workers |
| **并发限制** | `SubagentLimitMiddleware` 截断多余 `task` 调用 | `MAX_CONCURRENT_SUBAGENTS = 3`, 15分钟超时 |
| **事件协议** | `task_started`, `task_running`, `task_completed/failed/timed_out` | SSE 格式 |

### 2.5 工具系统 (`tools/`)

| 类别 | 工具 | 来源 |
|------|------|------|
| **配置工具** | 从 `config.yaml` 解析 | `resolve_variable()` |
| **MCP 工具** | 从 `MultiServerMCPClient` 懒加载 | 支持 stdio/SSE/HTTP |
| **内置工具** | `present_files`, `ask_clarification`, `view_image`, `setup_agent`, `update_agent` | `tools/builtins/` |
| **子 Agent 工具** | `task(task_desc, prompt, subagent_type)` | `subagent_enabled` |
| **社区工具** | Tavily 搜索, Jina AI 抓取, Firecrawl 爬虫, DuckDuckGo 图片搜索 | `community/` |
| **ACP Agent** | `invoke_acp_agent` 调用外部 ACP 代理 | 需要真实 ACP 适配器 |

### 2.6 MCP 系统 (`mcp/`)

| 特性 | 技术参数 |
|------|----------|
| **核心库** | `langchain-mcp-adapters` `MultiServerMCPClient` |
| **初始化策略** | 懒加载，首次使用 `get_cached_mcp_tools()` |
| **缓存策略** | mtime 文件变更检测 |
| **传输协议** | stdio (command+args), SSE, HTTP |
| **OAuth 支持** | `client_credentials`, `refresh_token`；自动刷新 + Authorization 头注入 |
| **配置位置** | `extensions_config.json` → `mcpServers` |

### 2.7 技能系统 (`skills/`)

| 特性 | 技术参数 |
|------|----------|
| **格式** | 目录含 `SKILL.md` + YAML frontmatter (name, description, license, allowed-tools) |
| **加载策略** | 递归扫描 `skills/{public,custom}/`，渐近加载 |
| **注入方式** | 按需注入到系统提示词 |
| **内置技能** | research, report-generation, slide-creation, web-page, image-generation, claude-to-deerflow |
| **安装 API** | `POST /api/skills/install` 接受 `.skill` ZIP 归档 |

### 2.8 模型工厂 (`models/factory.py`)

| 特性 | 技术参数 |
|------|----------|
| **函数** | `create_chat_model(name, thinking_enabled)` | 
| **支持 Provider** | OpenAI, Anthropic, DeepSeek, vLLM, Ollama, Codex CLI, Claude Code OAuth |
| **Thinking 支持** | `supports_thinking` + `when_thinking_enabled.extra_body` 配置 |
| **Vision 支持** | `supports_vision` 标志，自动注入 `view_image` 工具 |
| **vLLM 兼容** | `VllmChatModel` (继承 `ChatOpenAI`)，保留 vLLM `reasoning` 字段 |

### 2.9 嵌入式客户端 (`client.py`)

| 特性 | 技术参数 |
|------|----------|
| **类** | `DeerFlowClient` — 进程内访问，无需 HTTP |
| **流式** | `stream()` → yield `StreamEvent` (values/messages-tuple/custom/end) |
| **同步** | `chat()` → 累积流 delta 返回最终 AI 文本 |
| **管理方法** | `list_models/skills`, `update_skill`, `install_skill`, `upload_files`, `get_memory` 等 |
| **CI 验证** | `TestGatewayConformance` 验证所有 dict 返回方法与 Gateway Pydantic 模型一致 |

### 3.2 IM 渠道系统 (`channels/`)

| 渠道 | 传输协议 | 消息模式 | 技术参数 |
|------|----------|----------|----------|
| **Telegram** | Bot API long-polling | `runs.wait()` → 最终响应 | 简单易配 |
| **Slack** | Socket Mode | `runs.wait()` → 最终响应 | 需 Bot Token + App Token |
| **Feishu/Lark** | WebSocket | `runs.stream()` → 增量卡片更新 | 同卡片 patch，`update_multi=true` |
| **WeChat** | 腾讯 iLink long-polling | — | 支持二维码引导 + 持久化 state |
| **WeCom** | WebSocket | — | 支持文本/图片/文件双向 |
| **DingTalk** | Stream Push WebSocket | AI Card streaming | 可选 AI Card 打字机效果 |

**架构**：Channel impl → `MessageBus.publish_inbound()` → `ChannelManager._dispatch_loop()` → Gateway LangGraph-compatible API

### 3.3 认证与安全 (`gateway/auth/`)

| 组件 | 功能 | 技术参数 |
|------|------|----------|
| **JWT** | 令牌处理 | Cookie-based，CSRF Double Submit Cookie |
| **AuthMiddleware** | 请求认证中间件 | fail-closed，非公开路由需有效 JWT |
| **CSRFMiddleware** | CSRF 防护 | Origin 校验 |
| **LangGraphAuth** | LangGraph API 认证 | 内部服务自动注入 |
| **InternalAuth** | 内部服务认证 | process-local internal auth |
| **LocalProvider** | 本地认证提供者 | 用于开发/单用户模式 |

| E2E tests | PR (frontend 变更时) | Playwright Chromium |

| 文件 | 优先级 | 说明 |

## 六、关键设计决策

1. **Harness/App 拆分** — 框架层 (`deerflow.*`) 与应用层 (`app.*`) 严格分离，`test_harness_boundary.py` CI 强制防火墙规则。
2. **Cookie 认证** — 采用 Cookie-based JWT + CSRF Double Submit Cookie，专为浏览器客户端设计，非 Bearer Token。
3. **LangGraph 平台兼容** — SSE 格式和 REST API 与 LangGraph Platform 一致，`useStream()` React hook 无需修改即可使用。
4. **Per-User 数据隔离** — memory/threads/agents 均按 `user_id` 隔离在文件系统路径和元数据中。
5. **配置热加载** — `config.yaml` 和 `extensions_config.json` 通过 mtime 检测自动重载，无需重启进程。
6. **Sandbox 虚拟路径** — Agent 看到的路径 `/mnt/user-data/*` 统一映射到 per-user per-thread 的物理目录。
7. **EAI-Flow 扩展 PostgreSQL** — 与核心 DeerFlow 分离，通过独立 Docker 服务 `eai-flow-postgres-ext:5434` 运行。

## 七、开发注意事项 (Gotchas)

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| `pnpm build` 失败 | 缺少 `BETTER_AUTH_SECRET` | 设置环境变量或 `SKIP_ENV_VALIDATION=1` |
| `pnpm check` 失败 | `next lint` 路径不兼容 | 使用 `pnpm lint && pnpm typecheck` 代替 |
| `make config` 报错退出 | 非幂等设计 | 使用 `make setup` 进行重配 |
| `make dev` 中断后异常 | 进程清理不完整 | 再次执行 `make stop` |
| `pnpm install` 失败 | 代理环境变量干扰 | 取消设置 `HTTP_PROXY`/`HTTPS_PROXY` |
| Windows 脚本失败 | 原生 cmd/PowerShell 不兼容 | 必须使用 Git Bash |
| 扩展模块 DB 连接失败 | 环境变量未设置 | 配置 `EXTENSIONS_DB_*` 环境变量 |
