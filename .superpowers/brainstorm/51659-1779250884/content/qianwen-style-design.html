项目概述

本项目旨在构建一套完整的智能化文档管理系统，通过AI技术提升文档创建、编辑和管理的效率。

系统采用前后端分离架构，前端使用 Next.js + React，后端使用 Python FastAPI。

技术架构

前端技术栈包括：

  • Next.js 16 (App Router)
  • React 19
  • Tailwind CSS 4
  • Shadcn UI

后端采用 LangGraph + FastAPI 构建，支持流式响应和工具调用。

技术架构

系统采用现代化的前后端分离架构：

前端技术栈：
• Next.js 16（App Router）— 服务端渲染与静态生成
• React 19 — 组件化 UI 开发
• Tailwind CSS 4 — 原子化 CSS 框架
• Shadcn UI — 高质量组件库

后端技术栈：
• LangGraph — AI Agent 编排框架
• FastAPI — 高性能异步 API 框架
• 流式响应支持（SSE）
• 工具调用与技能扩展系统

核心交互流程

快捷操作 选中编辑器文字 → 点击快捷标签（润色/扩写等） → 自动发送选中文字+操作到AI → AI回复
自定义指令 在输入框输入自定义文字 → 点击发送 → AI回复（可同时附带选中文字作为上下文）
替换操作 点击AI回复下方的「替换选中内容」→ 将AI生成的内容替换编辑器中选中的文字
多轮对话 用户可以追问、修改要求，AI 基于上下文继续回复。对话历史在面板中保留
推荐提示 空状态时显示2-3个推荐提示，点击直接发送（如「总结全文要点」「优化文档结构」）
新对话 点击 🔄 按钮清空对话历史，重新开始

设计要点

  • 编辑器和 AI 面板并排，同时可见（仿千问笔记）
  • 快捷操作标签始终可见，紧凑不占空间
  • 对话区域占据大部分面板空间
  • 每条 AI 回复支持 Markdown 渲染 + 替换/复制按钮
  • 底部输入框内嵌模型选择器
  • 空状态有推荐提示引导用户

技术实现要点

  • 对话历史用 useState 管理（不持久化）
  • Markdown 渲染可用 react-markdown
  • 流式响应（SSE）支持逐字显示
  • 选中文字通过编辑器 API 获取
  • 替换操作通过编辑器 API 完成
