Temporal Activities 定义

阶段管理 Activities
  • init_phase — 初始化阶段（创建章节、分配团队）
  • gather_phase_context — 收集阶段产物作为上下文
  • advance_phase — 推进阶段状态，更新 report_project.current_phase_node
AI 生成 Activities
  • start_ai_writing — 调用 DeerFlow Agent 撰写
  • parse_sources — 解析 AI 输出中的溯源标记
  • store_sources — 批量写入 content_sources 表
审核 Activities
  • create_review_assignments — 创建 phase_reviews 记录
  • check_reviews_complete — 查询是否所有审核已完成
  • handle_rejection — 处理退回，回退工作流到目标节点
通知 Activities
  • notify_phase_start — 通知团队阶段启动
  • notify_review_pending — 通知审核人有待审核
  • notify_workflow_complete — 通知经理工作流完成

Temporal Signals（外部事件驱动）

# 工作流通过 signal 接收外部事件，不做 DB 操作
@workflow.signal
async def phase_complete
(self, phase_node: str, context: dict):
"""人工/系统标记阶段完成，传入阶段产物"""
self.pending_phases[phase_node] = context
@workflow.signal
async def review_action
(self, review_id: str, action: str):
"""审核人通过/退回，触发等待条件检查"""
self.review_results[review_id] = action
@workflow.signal
async def ai_complete
(self, chapter_id: str, result: dict):
"""AI 生成完成回调"""
self.ai_results[chapter_id] = result
现有 project API 完全保留： CRUD、outline、chapters、members、start-writing、start-editing — 均不修改，新功能通过 /workflow/* 和 /phase-reviews/* 端点扩展
