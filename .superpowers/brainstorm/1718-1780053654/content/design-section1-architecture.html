核心设计原则

1. 工作流定义 vs 执行分离

DAG JSON 存在应用库，Temporal 只管执行。用户编辑的是 JSON，不是 Temporal 工作流代码。

2. 现有模型扩展而非替换

ReportProject 保留但增加 workflow_id。阶段从硬编码 6 步变为由工作流定义驱动。

3. Activity 做所有副作用

Temporal Workflow 内不做 DB 操作。所有 DB/AI/通知操作都在 Activity 中执行。

4. 渐进式部署

Temporal 可独立启停。工作流定义存在应用库，即使 Temporal 未启动也能查看配置。
