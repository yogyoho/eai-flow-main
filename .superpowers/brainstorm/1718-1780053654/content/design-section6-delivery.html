设计 Section 6: 分阶段交付计划

4 个 Phase，每个可独立上线，渐进式交付价值

Phase 1: 工作流引擎 + React Flow 编辑器 ~3 周
后端：
  • Docker 新增 temporal-auto-setup 容器
  • FastAPI lifespan 集成 Temporal Client + Worker
  • workflow_definitions 模型 + CRUD API
  • DynamicGraphWorkflow 解释器
  • Phase/Review/Merge Activity 实现
  • DAG 校验服务（环检测、孤立节点）
  • Signal 处理（phase_complete, review_action）
前端：
  • React Flow 编辑器（5 种节点类型）
  • NodePalette 拖拽面板
  • PhaseConfigPanel / ReviewConfigPanel
  • DAG 校验反馈 UI
  • 存为模板 / 加载模板
交付价值：项目经理可以可视化编排工作流，系统按 DAG 自动推进阶段。替代原有固定 6 阶段。
Phase 2: AI 内容溯源 ~2 周
后端：
  • content_sources 模型 + API
  • 溯源解析服务（解析 [source] 标记）
  • MCP write_chapter Activity 集成溯源解析
  • report-write Skill 更新（要求 AI 输出溯源标记）
  • 溯源缺失检测 API
前端：
  • BlockNote 自定义扩展：溯源标注渲染
  • SourceFootnote 脚注列表组件
  • SourceAnnotation 悬浮提示组件
  • 溯源统计面板（按类型/章节）
  • 溯源缺失预警提示
交付价值：每个数据/段落可追溯到知识库文档、法规、AI 模型或人工输入。审核人可以验证引用准确性。
Phase 3: 多人分章节/维度混合审核 ~2 周
后端：
  • phase_reviews 模型 + API
  • Review Activity 创建分配记录
  • Signal review_action 处理通过/退回
  • 退回时工作流回退到目标节点
  • 审核维度配置（技术/合规/语言/...）
前端：
  • PhaseReviewPanel 审核工作台
  • ReviewAssignmentDialog 分配对话框
  • ChapterReviewCard 章节审核卡片
  • DimensionReviewCard 维度审核卡片
  • 审核进度可视化
交付价值：大型报告审核不再是一个人通读全文，而是多人并行按章节/维度分工审核，效率提升数倍。
Phase 4: 工作流监控 + 条件分支 + 子工作流 ~2 周
后端：
  • Condition Activity 评估表达式路由
  • Child Workflow 支持（子项目作为子工作流）
  • 工作流状态查询 API（含 Temporal history）
  • 工作流暂停/恢复/取消 API
前端：
  • WorkflowMonitor 执行监控面板
  • PhaseStatusCard 状态卡片
  • TimelineView 时间线视图
  • ConditionNode 条件分支 UI
交付价值：项目经理实时监控工作流执行，条件分支支持差异化流程，子工作流支持子项目嵌套。完整工作流引擎能力闭环。
~9 周
