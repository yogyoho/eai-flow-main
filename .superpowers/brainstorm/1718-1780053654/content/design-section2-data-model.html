设计 Section 2: 数据模型

新增 4 张表，扩展现有模型，驱动三大新能力

模型关系图
NEW workflow_definitions — 工作流 DAG 定义（React Flow 生成）
id UUID PK
name VARCHAR "环评报告流程"
report_type VARCHAR 可选绑定
graph_json JSONB DAG节点+边
is_template BOOLEAN 预设模板
created_by UUID FK→users
↓ 1:N (一个定义可启动多个实例)
EXTEND report_projects
✅ id, name, report_type, status...
✅ current_stage (改为从 workflow 推导)
🆕 workflow_id FK→workflow_definitions
🆕 temporal_workflow_id VARCHAR
🆕 current_phase_node VARCHAR 当前节点
NEW phase_reviews — 阶段审核分配
id UUID PK
project_id FK→projects
phase_node VARCHAR DAG节点ID
chapter_id FK→chapters 可NULL
reviewer_id FK→users
review_type chapter|dimension
dimension VARCHAR 技术|合规|语言...
status pending|approved|rejected
comment TEXT
created_at TIMESTAMP
NEW content_sources — AI 内容溯源（段落级）
id UUID PK
chapter_id FK→chapters
block_index INT BlockNote块索引
source_type ENUM
source_ref VARCHAR 来源标识
snippet TEXT 引用片段
confidence FLOAT 0-1
metadata JSONB 扩展信息
created_at TIMESTAMP
source_type 枚举： knowledge_base rag_retrieval ai_generated human_written template external_data
现有表不变： project_chapters, project_members, approval_workflows, approval_records, collab_documents — 无结构变更
