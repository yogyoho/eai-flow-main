协作智能写作 — 松耦合架构设计

核心思路：项目管理是元数据层，报告编写复用 DeerFlow 现有能力

整体架构

用户视角（前端 3 个页面）
┌─ 项目管理页 → 项目Card列表 → 点击"进入项目"
→ 自动创建/跳转到绑定线程的 DeerFlow 对话页
└─ 项目文件视图 → 聚合项目下所有线程文件 → 查看/编辑
后端（纯 App 层，不修改 harness）
├─ report_projects 表 → 项目元数据 + 成员关联
├─ project_thread_bindings 表 → 项目×成员×线程 映射
├─ Gateway Router → 项目 CRUD + 文件聚合 API
└─ Prompt 注入 → 通过 thread_data 注入模板上下文

核心流程

1. 创建项目
选择领域 → 选择报告类型 → 选择模板 → 添加成员 → 生成项目 Card
2. 进入项目对话
成员点击 Card → 自动创建/绑定线程 → Prompt 自动注入模板上下文
→ 进入 DeerFlow 对话页，Agent 已知道"你在写 XX 类型的报告，模板是 YY"
3. AI 生成报告
成员在对话中要求 AI 生成报告内容
→ Agent 根据注入的模板上下文，按章节结构生成
→ 文件输出到线程目录（现有机制，零改动）
4. 文件共享
项目文件视图 → 聚合查询所有成员线程的文件目录
→ 虚拟展示，底层文件仍在各线程目录（零改动）
5. 合规校验
在项目页面一键触发 → 调用现有 ComplianceEngine
→ 结果展示在项目 Card 或详情页（复用 knowledge_factory）

数据模型

report_projects {
  id: UUID,
  name: String, // "XX项目环评报告"
  domain_id: String, // → extraction_domains.id
  report_type: String, // "环评"/"地质勘查"/"可研"
  template_id: UUID, // → extraction_templates.id
  status: String, // active/completed/archived
  created_by: UUID,
  members: [project_members]
}

project_members {
  project_id: UUID,
  user_id: UUID,
  role: String, // owner/editor/reviewer
  thread_id: String // 绑定的 DeerFlow 线程ID
}

不改动 harness 的关键点

Prompt 注入方式
方式1（推荐）：通过 thread_data 元数据注入
创建线程时在 metadata 中写入项目模板信息
→ ThreadDataMiddleware 已读取 thread_data
→ 只需在 App 层创建线程时写入，harness 层自动读取

方式2（备选）：通过 Skill 注入
创建一个 report-writing SKILL.md，包含模板上下文
→ 项目线程启用该 Skill
