不同角色进入项目后的体验差异，以及 Tab 可见性规则

🎯 角色默认页映射

角色 默认页 定位
👑 经理 (manager) 📊 仪表盘 全局管理者视角 — 看进度、管成员、控流程

📝 撰写人 (writer) 🎯 我的工作台 内容生产者 — 撰写自己被分配的章节

🏗️ 前端实现架构

// types.ts — Tab 配置与角色映射

interface TabConfig {
  id: string;
  label: string;
  icon: string;
  stages: number[];            // 在哪些 Stage 可见
  minPermission: string;       // 最低权限要求（null = 所有成员）
  defaultFor: string[];        // 哪些角色默认进入此 Tab
  accessLevel: "full" | "read" | "hidden";  // 按角色控制
}

✅ 关键设计点

  • TAB_REGISTRY 是唯一的 Tab 配置源 — 后端权限决定可见性，前端不硬编码
  • 默认页由角色决定 — editor/writer 进入「我的工作台」，manager 进入「仪表盘」
  • 审计人/批准人 在 Stage 5 时审核 Tab 自动出现并设为默认
  • 委托方 只能看仪表盘（只读）和审核状态，不能编辑任何内容
  • Stage 联动 — Tab 可见性同时受 Stage 和角色控制，两个条件都要满足
